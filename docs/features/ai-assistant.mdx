Conar's AI Assistant is a schema-aware conversational AI built directly into your database workflow. Unlike generic AI chatbots, the assistant has full context of your connected database — it knows your table names, column types, relationships, and more — so it can write accurate SQL, explain query behavior, and troubleshoot errors without you needing to paste schema definitions into every message. The assistant is available from the chat panel alongside the SQL Editor, and its capabilities extend throughout the app into the Table Browser and inline editor features.

<Note>
  The AI assistant only executes safe, read-only queries against your database when using its database tool. It will never run `INSERT`, `UPDATE`, `DELETE`, or `DROP` statements autonomously. Any data modifications go through the normal Table Browser or SQL Editor workflows, where you remain in full control.
</Note>

## Chat Interface

The AI chat panel lives alongside the SQL Editor in the Query view. Open it by clicking the **AI Assistant** button in the connection toolbar. The panel is resizable, so you can expand it for a longer conversation or collapse it when you want more space for the editor.

You can ask the assistant anything about your database in plain English:

- *"Which tables store customer order data?"*
- *"Explain what this query does and whether it could be optimized."*
- *"Write a query to find users who signed up in the last 7 days but haven't placed an order."*

The assistant remembers the context of your conversation within a session, so you can follow up with refinements like *"now filter that to only include users in the EU"* without repeating yourself.

## Enhance Prompt

Not sure how to phrase your question? The **Enhance Prompt** button in the chat panel rewrites your draft message to be clearer, more specific, and more likely to produce a great response from the AI.

Type your question in the input box, then click **Enhance** before sending. Conar will reformulate your prompt — expanding vague requests into precise ones and adding relevant context from your schema — and place the improved version back in the input for you to review and send. You're always in control of what gets sent.

Enhance Prompt is useful when:

- You know what you want but aren't sure how to phrase a technical request.
- Your initial question is broad and you'd like the AI to ask clarifying questions.
- You want to add schema context to a question automatically.

## Natural Language Filters

Instead of manually building filter conditions in the Table Browser, you can describe what you want in plain English and let the AI create the filter rules for you.

<Steps>
  <Step title="Open the Filters panel">
    In the Table Browser, click the **Filter** button in the toolbar to open the filter panel for the active table.
  </Step>
  <Step title="Describe your filter">
    Click **Ask AI** in the filter panel and type a natural language description of the rows you want to see — for example, *"orders placed in the last 30 days with a total over $500 that haven't shipped yet"*.
  </Step>
  <Step title="Review and apply">
    The AI translates your description into one or more structured filter conditions using the correct columns and operators. Review them in the panel, adjust any condition if needed, and click **Apply** to filter the table.
  </Step>
</Steps>

<Note>
  Free-tier accounts include a limited number of AI-generated filter uses per month. Upgrade to a paid plan for unlimited natural language filters. See [Plans](/account/plans) for details.
</Note>

## AI Data Seeding

The **Seed** feature populates a table with realistic, schema-consistent test data. Open any table in the Table Browser, click the **Seed** button in the toolbar, and specify how many rows you'd like to generate.

Conar intelligently inspects your column names and data types to automatically select appropriate generators for each column — producing proper names, plausible email addresses, realistic timestamps, and foreign key values that reference existing records in related tables. You can review and adjust the generator assigned to each column before inserting any rows.

Seeded rows are previewed before insertion, so you can inspect and adjust them before committing to the database.
